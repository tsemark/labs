# Labs Repository

A personal workspace for experimentation, learning, and documenting experiences throughout my career journey.

## Overview

This repository serves multiple purposes:
- **Learning Diary**: Document experiences, lessons learned, and insights gained over the years
- **Tool Playground**: Experiment with new tools, frameworks, and technologies
- **Tutorial Documentation**: Create and maintain tutorial documentation for reference
- **Environment Templates**: Quick-start templates for spinning up POC environments and demo applications

## Repository Structure

```
labs/
├── DevOps/          # Infrastructure as Code, automation, and DevOps practices
│   ├── docs/        # Documentation, migration plans, post-mortems
│   ├── k8s/         # Kubernetes cluster configurations
│   ├── packer/      # Image building configurations
│   └── terraform/   # Infrastructure provisioning templates
│
└── QA/              # Quality Assurance and testing resources
    ├── automation/  # Test automation frameworks and samples
    └── docs/        # Test plans, bug reports, and QA documentation
```

## Key Sections

### DevOps
- **Kubernetes**: EKS and Kops cluster configurations with Terraform
- **Terraform**: AWS infrastructure templates for EC2, ECS, self-hosted runners, and more
- **Packer**: Custom AMI builds for various use cases
- **Documentation**: Migration plans, cost-saving strategies, and post-mortem reports

### QA
- **Test Automation**: Sample test frameworks (pytest, Playwright)
- **Documentation**: Test plans, bug reports, and testing methodologies

## Usage

Each subdirectory contains its own README with specific setup instructions and usage guidelines. Refer to individual project READMEs for detailed information.

## Purpose

This repository is designed to:
- Capture knowledge and best practices
- Provide reusable templates for common infrastructure patterns
- Serve as a reference for future projects
- Document real-world experiences and solutions
